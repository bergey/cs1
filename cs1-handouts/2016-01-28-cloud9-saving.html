Often when writing code, we would like to save our work so we can go back to some previous version of the code. Many of you have already told me, "This was working, now it doesn't, and I don't know what I changed." The longer you program, the more this will happen. Follow these instructions to save steps along the way. You will also be handing in your project this way.

The first 3 steps are needed once per project. The rest are needed each time you save a version you may go back to.

  1. Login to Cloud 9, and open your Dictionaries project
  2. In the Cloud 9 menu bar, select the Cloud 9 menu (to the left of File) and the Preferences menu item.
  3. In the Preferences tab, open the Experimental list, and turn on Changes Panel
  4. You should now see a Changes tab on the far left of the screen, vertically. Select this tab.
  5. Below the Cloud 9 menu is a Commit menu. Click the arrows to the right of Commit, and select Add All
  6. Click Commit
  7. Enter a description of your changes, like "wrote parseDate". Then click Commit
