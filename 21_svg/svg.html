We can describe images with HTML also. When we've learned this image language, we'll also be able to write Javascript to make more elaborate images. The image language is called SVG (short for Scalable Vector Graphics) and every image is inside an SVG tag.

If you want to make something I haven't taught, Mozilla has all the details.

Circles

We'll start with a single circle. A circle is described by its radius and the position of its center. If you leave off any of these 3 required attributes, most browsers won't draw the circle at all. All dimensions & coördinates are in pixels. The center position is described by the X (horizontal) position and the Y (vertical) position. (0,0) is the upper left, and coördinates increase to the right and down. I often write coördinates as (x,y), but SVG doesn't use that notation.

Rectangles

Circles are not the only shape we can make. SVG describes rectangles with the rect tag. A rect has a coördinate for the upper left corner, a width, and a height. For now, we will only draw rectangles with vertical and horizontal sides.

Transformations and Groups

Often the easiest way to describe a shape is in terms of some other shape and a transformation. For instance, if we have several copies of the same shape, instead of calculating coördinates for every copy, we can copy and paste the same SVG tags, and wrap each copy in a transformation describing how much to translate (move) the copy. We'll write our transform attributes on g tags. g is for group. g is a bit like div — it doesn't draw anything by itself, but it helps us group other elements to put them where we want.
